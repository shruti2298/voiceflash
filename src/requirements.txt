Backend Engineering Assignment

Memory Card Voice Bot using Pipecat
Objective
Build a voice-based Memory Card bot using the Pipecat framework. The bot should conduct a memory game with a user over a voice call. It speaks a sequence of words/cards, the user repeats the sequence back, and the bot evaluates whether the response is correct.
The game should get progressively harder as the user clears rounds.
What You Need to Build
Voice Bot Backend: A Pipecat-based pipeline that starts a memory game, speaks a sequence to the user, listens to the user repeat it, validates the response, updates the game state, and moves to the next round or ends the game.
Backend APIs and Database: Store game sessions, rounds, user responses, scores, and session status in a database. Create APIs for starting a session, fetching game state, ending a session, and viewing recent scores or leaderboard.
Caching: Use caching for some meaningful part of the system, such as active session state, current round state, leaderboard, or recently used sequences.
Frontend: A minimal web UI to start/join the memory game session and display game state such as current score, current round, session status, and final result.
Technical Requirements
Use Pipecat as the core framework for the voice pipeline.
Implement proper turn-taking: the bot should wait for the user to finish repeating the sequence before evaluating.
Handle interruptions cleanly. If the user interrupts the bot mid-speech, the bot should recover gracefully. Demonstrate this in the video.
The bot should have an engaging and human-like game-host behavior.
Persist session, round, response, and score data in a database.
Expose backend APIs for session and score-related data.
Use caching for at least one meaningful backend flow.
Avoid double-scoring the same user response.
The word/card list can be hardcoded or seeded.
